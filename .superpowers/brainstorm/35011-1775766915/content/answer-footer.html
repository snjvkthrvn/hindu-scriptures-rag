How should the answer footer be arranged?

Each answer needs to hold citations (verses cited), actions (copy, regenerate, go deeper, simpler), and follow-ups (suggested next questions). These compete for the same real estate. Three arrangements — all shown in light theme for clarity. Same answer body in each.

A · Stacked

Citations inside the bubble (as clickable chips at the bottom), action row just below, follow-ups live OUTSIDE the bubble as standalone chip-buttons. Clear hierarchy: answer → citations → actions → "what's next". Feels like a well-structured document footer.

Pros: clearest hierarchy, follow-ups feel like invitations, not part of the answer. Cons: takes the most vertical space.

B · Compact Hover

Actions live as small corner pills that appear on hover (like ChatGPT/Claude). Citations are collapsed into "▸ 4 Sources" — click to expand a drawer. Follow-ups become short saffron chips, not full sentences. Most compact; lets the answer breathe.

Pros: minimal visual weight, answer dominates. Cons: discoverability hit — hover doesn't exist on mobile, so actions need a permanent mobile alternative. Citations being collapsed means users might miss them.

C · Parchment Sidecar ★

Answer bubble stays clean (no footer inside). Below it, a distinct parchment panel holds citations and follow-ups together — framed like a companion note. Actions sit as a minimal text-link toolbar underneath. Feels like an answer with a small footnote card attached.

Pros: the answer is untouched/sacred, metadata has its own room to breathe, works identically on mobile. Cons: more vertical real estate, parchment-on-parchment needs careful contrast tuning.
My recommendation

C · Parchment Sidecar. Two reasons: (1) it keeps the answer bubble clean — the sacred text and prose don't get cluttered by UI chrome, which matters a lot for the Sacred Interface direction. (2) it works the same on desktop and mobile (no hover dependency). Actions as text-links are unobtrusive but fully tappable. Option A is my runner-up if you want actions to feel more "button-y". Option B is the most modern/compact but has real discoverability issues on touch devices.
