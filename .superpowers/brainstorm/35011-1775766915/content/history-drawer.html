A · Clean List

Simple, ChatGPT-like. Header with Om + title + close. Prominent "+ New Conversation" button. Threads as single-line titles grouped by Today / Yesterday / Last Week / Older. Current thread highlighted with a saffron left border. No search, no previews — just threads.

Best for: users who have few threads and know them by title.

B · Rich Cards ★

Header + search box + New button. Each thread is a card with a preview line (first few words of the last answer), scripture tags pulled from citations, and a message count. Grouped by time. Active thread gets a saffron border all around.

Best for: users with many threads who need to scan and remember by content, not just title. The scripture tags tie beautifully into the Reading Companion direction.

C · Minimalist Ribbon

Narrower drawer. No search, no previews, no date groupings. Just "+New" and a flat chronological list of thread titles. Smallest footprint, least visual noise.

Best for: users who treat each session as ephemeral and mostly use "+New" — history is a safety net, not a workspace.
My recommendation

B · Rich Cards. Reason: for a Reading Companion product, users will build up a real library of threads over weeks. Previews + scripture tags transform the history drawer from a "switch list" into a personal reading archive — which is exactly the Companion direction's promise. Search is essential once you have more than ~15 threads. Option A is cleaner but has less memory-jogging power; Option C is too austere for daily return.

Note: this does require a small change to the data model — we'd need to store a preview snippet and scripture tags per thread in localStorage. Very small lift.
